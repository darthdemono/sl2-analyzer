What it supports

Drop a save from any of these. The game is detected from the file — no need to say which. Each renders in its own status screen.

  • Dark Souls Remastered — full: identity, class, all attributes, souls, humanity, inventory, held boss souls.
  • Dark Souls: Prepare to Die — full, same as Remastered.
  • Dark Souls II: Scholar of the First Sin — the deepest read: attributes, class, covenant, gender, hollowing, deaths, play time, full inventory with infusions and reinforcement, plus bonfires discovered and bosses defeated from the world block.
  • Dark Souls III — full: attributes, souls, inventory, held boss souls, bosses defeated.
  • Elden Ring — identity, attributes and runes in full; owned weapons, armour and ashes named; held remembrances and the bosses they prove. The rest of the held inventory (talismans, spells, goods) is not read yet, so the item list is partial.

Every game degrades a slot to "inventory only" rather than print a stat it cannot verify.

Not supported: vanilla Dark Souls II (pre-SOTFS) — its encryption key is not public.

How it works

A .sl2 is a FromSoftware save: one packed archive of blocks, most of them encrypted, one per character plus a few for the world. This tool unpacks the archive, decrypts each block with the key the game itself ships, and reads the fields off known positions. No upload and no server. The file is parsed in your browser and forgotten the moment you close the tab.

Every game hides its data differently, so each is read differently.

  • Dark Souls II (SOTFS): unlocked with its AES key, then read at fixed offsets. Bonfires lit and bosses killed come from a second world block the game keeps apart from the character.
  • Dark Souls Remastered / Prepare to Die: the character block moves as the save grows, so it is found by a marker that always sits beside it (Remastered) or by the name text itself (PtDE).
  • Dark Souls III: the stats shift between patches, so the block is not read from a fixed spot. It is found by content: the nine numbers that add up to your level. The game follows that rule, so a wrong match is close to impossible.
  • Elden Ring: the same trick with eight numbers, and every owned item read straight from the game's item array.

Two rules it never breaks. First, it never prints a number it cannot verify. If a stat block does not check out, the slot drops to "inventory only" and the attributes are simply gone. A missing field beats a wrong one. Second, the progress it shows is a floor, not a ceiling. It counts a boss dead only when the save still proves it: the soul in your pack, a defeat flag, or a point past it you could not have reached otherwise. A boss whose soul you already spent, with no flag and no gate, stays off the list. It under-claims on purpose.
